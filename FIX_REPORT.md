# 修复报告

> 日期: 2026-02-03

## 覆盖范围
- 严重问题（崩溃/数据丢失风险）
- 资源泄漏问题
- 空指针风险
- 异常处理不当
- 线程安全问题
- 逻辑错误

## 关键修复摘要
1. 运行列表索引越界不再 `std::exit()` 退出，改为提示并安全返回。
2. 加载提示窗口生命周期改为堆对象托管并在所有路径关闭，避免悬空引用。
3. 单实例本地服务改为栈对象管理，避免内存泄漏。
4. RPC 端口检测读取输出缓冲，避免进程阻塞。
5. 日志输出统一安全入口，避免 m_logTextEdit 空指针。
6. 节点信息 JSON 解析失败累计阈值后暂停定时更新，避免刷屏。
7. 线程相关变量改为原子/互斥保护，减少竞态风险。
8. 白名单配置恢复顺序修正，防止加载时数据/状态错位。
9. 启动日志对话框改为栈对象，避免延时销毁带来的误判。
